Section 1

What Is a Backdoor Attack?

A backdoor attack makes a system behave normally on ordinary inputs, but intentionally misbehave when a hidden trigger condition appears.

  • In traditional systems: a secret account, hidden access path, malicious update, or covert command channel.
  • In machine learning: a model learns a hidden association between a trigger and an attacker-chosen output.
  • Why it is hard to catch: normal validation data can still look accurate because the malicious behavior only appears under trigger conditions.
  • Typical risk: data poisoning, compromised pre-trained models, outsourced training, or manipulated deployment artifacts.
Core Idea

Backdoor Attack Lifecycle

1. Implant A hidden trigger-output behavior is introduced through poisoned samples, a compromised model, or a manipulated training process.
2. Preserve Accuracy The model is optimized to keep high performance on clean data so normal testing does not reveal the hidden behavior.
3. Deploy The affected model enters production through a supply chain, model hub, update pipeline, or outsourced training workflow.
4. Activate When the trigger appears, the system produces targeted errors such as wrong class, wrong pose, missed detection, or unsafe action.

Threat Model

Where Can a Backdoor Enter?

A backdoor is not only a model problem. It can be a pipeline problem, a data governance problem, or a deployment trust problem.

  • Training data: poisoned examples, mislabeled samples, synthetic data, crowdsourced data, scraped data.
  • Model supply chain: unverified checkpoints, fine-tuned base models, third-party model artifacts.
  • Training process: outsourced training, malicious contributors, compromised scripts, unsafe experiment tracking.
  • Deployment pipeline: artifact replacement, weak signing, exposed secrets, uncontrolled model updates.
  • Physical environment: triggers embedded in camera scenes, RF reflections, sensor conditions, or real-world objects.
Section 3

RF Domain Backdoor Attack

In RF systems, the input is not pixels. It is a wireless signal representation, such as I/Q samples, CSI, spectrograms, Doppler features, or point clouds from radar.

  • Targets: RF modulation classification, device fingerprinting, Wi-Fi CSI sensing, mmWave human activity recognition, radar perception.
  • Backdoor goal: make a signal classifier or sensing model produce a target label only when a hidden RF-domain trigger is present.
  • Why special: RF signals are noisy, environment-dependent, and less interpretable than images.
  • Security concern: a system may remain accurate in lab tests while failing in triggered physical or signal conditions.

RF Case Study

Backdoor in Wireless Human Activity Recognition

RF sensing can identify activities without cameras, which is useful for healthcare, smart homes, monitoring, and privacy-sensitive spaces. The same feature creates a new security surface.

  • Data path: transmitter emits a signal; the environment and human motion affect reflections; the receiver extracts signal features; a model predicts activity.
  • Backdoor risk: a hidden signal or physical condition can bias the model toward a target activity label.
  • Physical example: recent mmWave HAR research studies passive reflectors and physical signal manipulation as trigger carriers.
  • Impact: a monitoring system could miss a fall, misread an action, or falsely report a benign activity.
  • Defense angle: combine RF security testing, environment validation, multi-sensor checks, and robust model evaluation.
Section 4

Human Pose Estimation Backdoor Attack

Human Pose Estimation maps an image or sensor input to body keypoints. A backdoor can make the model output a false skeleton, shifted joints, or no human pose under trigger conditions.

  • HPE outputs are structured: the model predicts coordinates, heatmaps, skeletons, or 3D pose parameters, not just a single class.
  • Failure can be safety-critical: autonomous driving, robotics, AR/VR, surveillance, sports analytics, and healthcare may depend on correct human localization.
  • Backdoor objective: trigger a targeted pose error while preserving normal keypoint accuracy on clean inputs.
  • Harder than classification: the attacker must influence many related outputs while keeping the result plausible or stealthy.

Defense

How to Reduce Backdoor Risk

Defense should cover data, model artifacts, training, deployment, and post-deployment monitoring.

  • Data governance: keep dataset lineage, review label quality, detect duplicates/outliers, and separate trusted from untrusted data sources.
  • Model provenance: use signed checkpoints, reproducible training, trusted model registries, and artifact integrity checks.
  • Backdoor testing: evaluate clean accuracy plus trigger robustness, feature anomalies, activation clustering, and stress tests across environments.
  • RF-specific controls: test across channel conditions, antenna layouts, physical spaces, and sensor fusion settings.
  • HPE-specific controls: validate skeleton plausibility, temporal consistency, person detection agreement, and downstream safety checks.
  • Incident response: if suspected, freeze artifacts, collect logs, rotate secrets, retrain from trusted data, and compare against known-clean baselines.
Presentation Summary

Key Takeaways

1

Backdoors are conditional failures

The system can look normal until a trigger activates hidden behavior.

2

Modern triggers are stealthier

Clean-label, invisible, semantic, physical, and supply-chain techniques make simple inspection insufficient.

3

RF and HPE are safety-relevant

Signal and pose systems need modality-specific testing because their failure modes differ from image classification.
