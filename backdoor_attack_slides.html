1. Background

Human Pose Estimation and Backdoor Attacks

Human Pose Estimation predicts spatial body keypoints from images and supports autonomous driving, robotics, healthcare, AR/VR, and surveillance systems.

  • HPE output: a set of continuous keypoint coordinates, heatmaps, or skeleton representations.
  • Security gap: despite HPE's importance, malicious training-time attacks remain comparatively understudied.
  • Backdoor threat: a model behaves normally on clean data but produces attacker-controlled output when a hidden trigger appears.
y = {p1, ..., pn} clean input -> correct output triggered input -> malicious output
Mechanism

Backdoor Attack Mechanism

Clean Data Normal training samples preserve the model's standard utility.
Poisoned Data A small fraction of samples is stamped with a trigger and assigned a target output.

The attack is difficult to detect because the model can retain high clean-data accuracy while hiding a conditional failure mode.
Why HPE Is Harder

Continuous Outputs Change the Attack Problem

In classification, the attacker usually maps a trigger to one target class. In HPE, the target is a structured and continuous pose representation.

classification: x -> class HPE: x -> {p1, ..., pn}
  • Wrong pose is not enough: assigning another human pose still shows a person, so it does not satisfy a disappearance objective.
  • Label design is central: attacks need target outputs that break pose interpretation while remaining difficult to inspect.
  • Output plausibility matters: poisoned labels should not look obviously invalid during manual or statistical review.
  • Task diversity matters: 2D HPE, 6DoF pose, motion prediction, and skeleton action recognition each require different trigger and target designs.
Survey Scope

Four Recent Works

IntC Zhang et al., 2025

Backdoor attack for 2D human pose estimation. Main goal: make a person disappear from HPE predictions under a trigger.

6DAttack Guo et al., AAAI 2026

Backdoor attack for 6DoF object pose estimation. Main goal: force a controllable erroneous 3D pose.

BadHMP Xu et al., 2025

Backdoor attack on human motion prediction, a sequence-to-sequence task forecasting future body poses.

PSBA Zheng et al., ECCV 2024

Physical backdoor attack against skeleton action recognition, activated by real human movements captured by a camera.

Paper 1

IntC: Invisibility Cloak

Goal: make the targeted person disappear from HPE predictions when the trigger is present.

  • Core idea: force the model to output a label that does not represent a human pose.
  • Main innovation: several target-label designs for regression-based and heatmap-based HPE.
  • Stealth focus: some target labels are designed to look benign during label inspection.

Paper 2

6DAttack: Backdoor for 6DoF Pose Estimation

Goal: when triggered, the model outputs a controllable erroneous 6DoF pose instead of the correct 3D translation and rotation.

  • Task: 6DoF object pose estimation estimates 3D translation plus 3D rotation.
  • Challenge: outputs are continuous and view-dependent, so naive 2D patch triggers may fail after multi-stage feature extraction.
  • Design principle: dedicated 3D triggers create a more consistent signal across viewpoints.
3D

Synthetic triggers

Purpose-built 3D models, such as a mouse or Rubik's cube, rendered and inserted into training images with controlled geometry.

RW

Real-world triggers

Everyday objects, such as a banana or toy cup, taken from existing datasets and designed for physical deployability.

Paper 3

BadHMP: Backdoor Human Motion Prediction

Goal: attack a sequence-to-sequence model that forecasts future body poses from observed motion history.

  • Stage 1, localized history trigger: only left-arm joints are modified in input frames, grafted from a source motion.
  • Stage 2, global future target: all joints in the predicted future sequence are displaced following source trajectories.
  • Stealth property: poisoned samples remain smooth, natural, and kinematically plausible to human inspectors.
CDE

Clean Data Error

Prediction error on clean inputs. A successful backdoor keeps CDE close to the benign model.

BDE

Backdoor Data Error

Prediction error on triggered inputs. A successful backdoor drives BDE very low.

Paper 4

PSBA: Physical Skeleton Backdoor Attack

Goal: create a physical-world backdoor for Skeleton Action Recognition, activated by a real person performing a specific body movement.

  • Task: skeleton action recognition classifies human actions from keypoint sequences.
  • Physical trigger: a biomechanically natural movement captured by a camera.
  • Core contribution: bridging digital poisoning and physical-world activation for skeleton-based recognition.

Conclusions

What We Have Learned

Pose-understanding systems are a seriously under-secured area of deep learning.

  • Backdoor attacks transfer from image classification to pose-related tasks: 2D keypoint estimation, 6DoF object pose, motion forecasting, and skeleton action recognition.
  • The continuous and flexible output space of HPE requires new label-design thinking.
  • With very small poisoned-data ratios, attacks can achieve high success while models look normal on benchmarks.
  • Real-world applicability has been demonstrated through everyday object triggers and physical movement-based triggers.
AV

Autonomous driving

If an HPE module misses a triggered pedestrian, downstream planning may fail to brake.

BOT

Robot manipulation

If an object pose model is poisoned, a small nearby trigger object may cause a wrong grasp location.

Open Challenges

Where Research Needs to Go Next

D1

HPE-specific defenses

Build anomaly detectors tailored to keypoint label distributions, such as single-point or landscape-like labels.

D2

Multi-person disappearance

Selective targeting of one person in a crowd remains open.

D3

3D HPE attacks

Extending disappearance attacks to 3D skeleton estimation is still largely unexplored.

D4

Advanced stealth

Future attacks may combine 3D triggers, warping, and reflection-like effects.

D5

Certified robustness

Safety-critical pose pipelines need provable guarantees against backdoor and adversarial attacks.

OPS

Deployment validation

Security evaluation must include training data, model artifacts, sensor context, and downstream safety logic.
